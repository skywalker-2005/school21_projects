# MVC

`MVC (Model-View-Controller)` - Это паттерн проектированиия помогает разделить приложение на три основных компонента, улучшая структуру кода, поддерживаемость и масштабируемость.

Архитектура MVC не относится к какому-либо конкретному языку программирования или фреймворку, но модель `MVC` широко используется для целей разработки веб-приложений, например для платформы `ASP.net`.

## Модель (Model)

Модель - это место, где доступна вся логика, связанная с данными, такая как схемы и интерфейсы проекта, базы данных и их поля. Она представляет данные пользователю. Этот уровень определяет, где хранятся объекты данных приложения. В контексте `nestjs` или `nodejs` это могут быть классы или объекты, которые представляют сущности в приложении.

## Представление (View)

Представление содержит пользовательский интерфейс приложения. Это визуальное представление модели, которое создает интерфейс для отображения фактических выходных данных пользователю. В контексте В контексте `nestjs` или `nodejs`, `View` может представлять данные, отправляемые клиенту в ответ на запросы. Это обычно данные, форматированные в JSON или другие форматы (`html`) для передачи клиенту.

## Контроллер (Controller)

Контроллер - это уровень, который действует как мозг всей системы MVC. Контроллер содержит логику, связанную с бизнесом, и обрабатывает входящие запросы. Фактически, контроллер - это интерфейс между представлением и моделью. В `Nest.js` контроллеры выполняются с использованием декораторов

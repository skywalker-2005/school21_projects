## Chapter II
## Общая информация

**Веб-приложение** — это программное приложение или сервис, доступный через веб-браузер, которое обычно взаимодействует с пользователем посредством веб-интерфейса. Веб-приложения обладают динамическим содержимым и могут предоставлять различные функции, такие как обработка данных, совместная работа, обновление в режиме реального времени и другие формы взаимодействия.

### Темы для изучения:
- Веб-приложение;
- Node.js;
- Nest.js;
- API;
- Алгоритм «Минимакс»;
- MVC.

## Chapter III
### Проект: Крестики-Нолики
Рассматриваются основы создания веб-приложения в среде `Node.js` и фреймворка `Nest.js`.

Для выполнения заданий необходимо использовать актуальную версию `NodeJS` и `NestJS`. А также `IDE`, например, `VSCode`.

Проект создаётся один раз и используется для всех последующих заданий.

## Задание 0. Создание проекта и установка зависимостей

- Если, у тебя ещё не установлен `NodeJS` или `NestJS`, то небходимо:
  - Установить `Nodejs`: можно воспользоватся специальным менеджером `nvm` или скачать с официального сайта;
  - Установить `Nestjs`: нужно в терминале ввести команду `npm i -g @nestjs/cli`.
- Выбери каталог для хранениия проекта.
- Войди через терминал в этот каталог и создай проект с помощью командной строки nest: `nest new tic-tac-toe-api`.
- Запуск осуществляется согласно установленной конфигурациии запуска в `package.json`, например, `npm start` или `npm run start:dev`.

## Задание 1. Создание структуры проекта
- Структура проекта должна иметь следующую архитектуру: modules (модули), models (модели), services (сервисы), контроллеры (controllers), а также включать следующие слои: web, domain, datasource.
- Каждый слой является отдельным модулем.
- Каждый модуль может содержать другие модули.
- Слой web должен включать в себя как минимум пакеты model, controller, mapper для взаимодействия с клиентом.
- Слой domain должен включать в себя как минимум пакеты model, service для реализации бизнес-логики приложения.
- Слой datasource должен включать в себя как минимум пакеты model, repository, mapper для реализации работы с данными (например, с базой данных).

## Задание 2. Реализация domain-слоя
- Опиши модель игрового поля в виде целочисленной матрицы.
- Опиши модель текущей игры, у которой есть UUID и игровое поле.
- Опиши интерфейс сервиса, у которого есть методы:
  - Метод получения следующего хода текущей игры алгоритмом Минимакс;
  - Метод валидации игрового поля текущей игры (проверь, что не изменены предыдущие ходы);
  - Метод проверки окончания игры.
- Модели, интерфейсы, реализации должны находиться в разных файлах.

## Задание 3. Реализация datasource-слоя
- Реализуй класс-хранилище для хранения текущих игр.
- Опиши модели игрового поля, текущей игры.
- Реализуй мапперы domain<->datasource.
- Реализуй репозиторий для работы с классом-хранилищем, у которого есть методы:
  - метод сохранения текущей игры;
  - метод получения текущей игры.
- Создай класс, реализующий интерфейс сервиса и принимающий в качестве параметра репозиторий для работы с классом-хранилищем.
- Модели, интерфейсы, реализации должны находиться в разных файлах.

## Задание 4. Реализация web-слоя
- Опиши модели игрового поля, текущей игры.
- Реализуй мапперы domain<->web.
- Реализуй контроллер, имеющий метод POST /game/{UUID текущей игры}, который отправляет текущую игру с обновленным игровым полем пользователем и получает в ответ текущую игру с обновленным игровым полем компьютером.
- Если отправлена некорректная текущая игра с обновленным полем, должна вернуться ошибка с описанием.
- Должна быть поддержка нескольких игр одновременно.
- Модели, интерфейсы, реализации должны находиться в разных файлах.

## Задание 5. Регистрация модулей и запуск приложения
- Зарегистрируй модули (слои) в глобальном модуле.
  - Определи параметры для импорта и экспорта.
  - Опиши настройку маршрутизации.
  - Определи запуск приложения.

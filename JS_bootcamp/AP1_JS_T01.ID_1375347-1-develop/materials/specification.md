# ECMAScript Specification

`ECMAScript` (abbreviated as ES) is a **specification** — a set of rules and guidelines that a language must follow in order to be considered ECMAScript-compliant.

`JavaScript` adheres to this specification.

The difference between them lies in purpose:

- **ECMAScript documentation** explains how to design a language that belongs to the ECMAScript “family.”
- **JavaScript documentation** explains how to use this specific language.

The ECMAScript “scripting language family” includes not just JavaScript, but also languages like **ActionScript**, **TypeScript**, and formerly **CoffeeScript**.\
Instead of describing each language individually, the ECMAScript specification outlines what they all have in common.

Because JavaScript follows the ECMAScript standard, all major changes and additions are documented in the specification.\
This helps developers understand where and how to track the evolution of the language and its features.

**Modern Specification Versions**

The modern and recommended baseline version of ECMAScript begins with **`ES2015`** (also known as **`ES6`**).\
Since 2015, the ES specification has been updated **annually**.\
Each year brings improvements, refinements, and new features that eventually make their way into JavaScript and are adopted by browsers.

**ES2015: A Turning Point**

The **ES2015** update marked a major milestone in JavaScript’s evolution:

- It was the **first update in six years**,
- And it introduced several foundational features:
  - **Arrow functions** — which preserve this context
  - **Class syntax** — syntactic sugar for OOP
  - **const and let** — for variable declarations
  - **Default function parameters**
  - **Object destructuring**
  - **Import/export modules**
  - **Promises**
  - ...and more
